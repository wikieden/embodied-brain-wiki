# 记忆机制 (Memory Mechanisms)

> 机器人如何存储、组织、检索和更新经验。记忆是具身大脑的基础组件——没有记忆，机器人就只是一个每次都从零开始的反射系统。

---

## 四种记忆类型

受启发于认知心理学的分类，机器人记忆可分为四个层次：

### 1. 工作记忆 (Working Memory)

**功能:** 当前任务的临时缓存，容量有限、持续时间短。

**例子:** “我正在把杯子放到桌子上，手里还拿着一把勺子”

**技术实现:**
- LLM 的上下文窗口 (context window)
- 感知-动作环的当前观察序列
- 注意力机制对重要信息的增强

**挑战:** 上下文窗口有限，如何在有限容量内保持最关键的任务信息？

### 2. 情节记忆 (Episodic Memory)

**功能:** 个人经历的事件序列，带有时间和空间信息。

**例子:** “昨天下午 3 点，我在厨房打破了一个碗，原因是台面太滑”

**技术实现:**
- **MemoryVLA** — 将历史感知序列存储为记忆嵌入，支持类似事件的检索
- **向量数据库 (Vector DB)** — 如 Chroma、Pinecone，将经验嵌入存储
- **图像/视频回忆** — 直接存储观察序列侜后续参考

**挑战:** 如何自动提取事件边界？如何检索“类似场景”而不是“关键词匹配”？

### 3. 语义记忆 (Semantic Memory)

**功能:** 通用知识库，不依赖于具体经历。

**例子:** “玻璃是脆的、水是流动的、烤箱是热的”

**技术实现:**
- 预训练知识（LLM 的参数即为语义记忆）
- 知识图谱 (Knowledge Graph)
- 概念网络和本体论

**挑战:** 如何将新经验提炼为通用知识？如何处理矛盾的知识？

### 4. 程序记忆 (Procedural Memory)

**功能:** 技能和习惯，通常是无意识的“肌肉记忆”。

**例子:** “怎么系鞋带”、“骑自行车”

**技术实现:**
- VLA 策略本身就是程序记忆（训练后的神经网络权重）
- 技能库 (Skill Library)
- 分层控制器中的低层反射

**挑战:** 如何在不破坏旧技能的前提下学习新技能？（灾难性遗忘）

## 记忆架构设计模式

### 模式 1: 外部记忆库 (External Memory Store)

机器人不自己存储，而是查询外部数据库。

- **向量检索:** 将当前观察嵌入，检索最相似的历史经验
- **图像存储:** 保存关键帧侜回顾
- **优点:** 简单、可解释
- **缺点:** 检索质量依赖嵌入模型，存储成本高

### 模式 2: 感知-认知双系统 (Perception-Cognition Dual System)

感知层处理原始观察，认知层维护抽象表征。

- **MemoryVLA** 是典范：VLA 处理当前观察，独立的认知模块管理长期记忆
- **优点:** 感知和记忆解耦，各自独立优化
- **缺点:** 双系统的协调复杂

### 模式 3: 统一波动网络 (Unified Latent Network)

所有信息都在同一个潜在空间中表征，无需明确分类。

- 类似人脑皮层的方式：同一组神经元即处理感知也存储记忆
- **优点:** 灵活、可能更接近生物机制
- **缺点:** 难以分析和控制

## 核心挑战

1. **灾难性遗忘 (Catastrophic Forgetting):** 学习新任务时覆盖旧知识
2. **记忆编码:** 如何将持续的感知流编码为可存储、可检索的表征
3. **时间检索:** “找出上周在厨房发生的事” — 需要时间索引
4. **记忆压缩:** 如何在有限存储中保持最多有用信息
5. **真实性验证:** 如何确保记忆的内容是准确的，而不是幻觉
